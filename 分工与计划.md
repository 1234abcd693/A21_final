# 船舶装备故障诊断智能问答系统 —— 5 周完整开发计划（双人协作）

> 本计划整合了 **溯源机制、防幻觉机制、C++ 极致性能、知识抽取流水线、U 盘同步、手动反馈调参** 等所有核心要求，并在 **5 周** 内实现可演示、可打包的桌面应用。  
> **角色分工**：  
> - **成员 A（系统集成）**：Electron + Vue3 前端、FastAPI 后端、RAG 检索与生成、LLM 集成（llama.cpp）、性能优化、API 暴露。  
> - **成员 B（知识与数据）**：知识图谱设计、Neo4j 数据管理、知识抽取（规则+人工）、文档解析、同步数据打包、反馈数据表。

---

## 二、详细周计划

### 第 1 周：基础框架与知识图谱初始

**目标**：搭建可运行的基础环境，核心组件互通。

#### 成员 A（系统集成）
- 创建 Electron + Vue3 + Vite 项目，主进程加载空白页面。
- 创建 FastAPI 项目（Python 3.10），实现 `/health` 接口。
- 下载 Qwen2-3B 4-bit GGUF 模型文件，编译 `llama.cpp` 得到 `llama-server.exe`。
- 启动 `llama-server`，编写 Python 调用封装（HTTP 流式请求）。
- 定义项目 API 接口文档（与 B 协商）。

#### 成员 B（知识与数据）
- 安装 Neo4j 绿色版，设计知识图谱本体（节点：故障现象、故障原因、维修步骤、设备类型；关系：CAUSED_BY, FIXED_BY, BELONGS_TO）。
- 编写脚本导入赛题提供的故障案例（或手工构造 50 个实体，10 个关系）。
- 实现基本 Cypher 查询（如 `MATCH (n) RETURN n LIMIT 25`）。
- 编写 `GET /symptoms` 所需查询（返回所有故障现象名称）。

#### 协作与交付
- 两人对齐 API：`/symptoms`, `/ask`（暂为 mock）, `/health`。
- 交付：可启动 Electron 空窗口；FastAPI 服务正常；Neo4j 有 50+ 实体；llama-server 能返回简单文本。

---

### 第 2 周：检索与问答基础

**目标**：实现混合检索和流式问答，答案带引用标记。

#### 成员 A
- 安装 Chroma，选择嵌入模型（`paraphrase-multilingual-MiniLM-L12-v2`）。
- 实现 BM25 索引（`rank_bm25`）和向量检索的两路融合（简单加权，初始 α=0.5, β=0.5）。
- 实现 `/ask` API：
  - 接收问题 → 检索 top-5 片段（每个片段带元数据：文档名、页码）。
  - 构造 prompt，要求模型输出结构化答案并标注引用编号（如 `[1]`）。
  - 调用 `llama-server` 流式生成，通过 SSE 返回前端。
  - 生成完毕后，进行**引用完整性验证**（检查编号是否在检索结果范围内）。
- 实现简单精确缓存（内存字典）。
- 前端基础聊天界面（输入框、气泡、流式显示）。

#### 成员 B
- 协助 A 准备检索测试数据：将已有知识图谱中的“故障现象+原因+步骤”文本分块，存入 Chroma 并提供元数据。
- 确保每个片段携带正确的来源信息（文档名、页码）。
- 提供 10 条典型故障问题测试集，用于验证检索效果。

#### 协作与交付
- 交付：可通过前端或 Postman 调用 `/ask`，得到流式答案，答案中带有 `[数字]` 引用。

---

### 第 3 周：知识抽取与导入

**目标**：支持从 Word/PDF 中抽取知识并导入 Neo4j + Chroma。

#### 成员 B
- 实现文档解析器：`python-docx` 读 `.docx`，`pdfplumber` 读 `.pdf`，输出纯文本。
- 实现基于正则的抽取规则：
  - 匹配“故障现象：...”、“故障原因：...”、“维修步骤：...”等模式。
  - 输出候选三元组列表（现象、原因、步骤）。
- 提供 `POST /parse_document` API：上传文档，返回候选三元组（JSON）。
- 实现 `POST /confirm_import` API：
  - 接收用户确认的三元组。
  - 将实体/关系写入 Neo4j，同时将文本片段（带元数据）通过内部接口通知 A 写入 Chroma。
- 确保每个新入库的节点携带 `source_doc`, `source_page` 属性。

#### 成员 A
- 实现 `/parse_document` 和 `/confirm_import` 的路由封装（调用 B 的函数）。
- 提供前端上传文档页面（简单拖拽上传），展示候选列表，允许用户编辑/删除/确认。
- 实现前端调用 `/confirm_import` 并刷新故障现象列表（用于联想输入框）。
- 实现 `/symptoms` 接口（从 B 处获取数据，供联想输入框使用）。

#### 协作与交付
- 交付：用户可上传 Word/PDF，系统自动抽取候选，确认后知识入库；联想输入框可提示新导入的故障现象。

---

### 第 4 周：溯源增强、同步与反馈

**目标**：强化可靠性，实现 U 盘同步和手动反馈调参。

#### 成员 A
- **防幻觉增强**：
  - 在 `/ask` 生成后增加**事实一致性检查**：提取答案中的关键名词（用 jieba 分词），检查是否出现在检索片段文本中。若不通过，在答案末尾添加“⚠️ 部分信息未经源文档确认，请参考原始资料。”。
  - 前端展示引用悬浮：鼠标悬浮 `[1]` 时显示来源文档名和页码。
- **反馈调参**：
  - 实现 `/feedback` 路由（存储反馈到 B 提供的 SQLite）。
  - 实现 `/optimize` 路由：读取反馈数据，网格搜索最优 BM25/向量权重（α, β），更新配置文件。
  - 前端增加“👍/👎”按钮，以及“优化检索参数”按钮。
- **同步模块**：实现 `/export_sync`, `/import_sync` 路由（调用 B 的打包/解包函数）。
- **报告导出**：实现 `/export_report`，生成 Word 报告（含问题、答案、引用列表）。

#### 成员 B
- 设计 SQLite 反馈表 `feedback`（字段：id, question, answer, retrieved_docs_ids, rating, created_at）。
- 实现反馈存储函数供 A 调用。
- 实现打包函数：将 Neo4j 数据目录、Chroma 目录、SQLite 文件打包为 zip（导出时需先停止 Neo4j）。
- 实现导入函数：解压 zip，停止 Neo4j，覆盖目录，重启 Neo4j。
- 提供反馈数据查询接口（供 A 优化时获取正负例）。

#### 协作与交付
- 交付：答案有引用悬浮及低可信警告；可导出/导入知识包到 U 盘；用户可点赞点踩并手动触发参数优化。

### 第 5 周：性能优化、打包与文档

**目标**：满足性能指标，交付可执行程序及参赛材料。

#### 成员 A
- **性能优化**：
  - 调整 `llama-server` 参数：线程数（`-t` 物理核心数），批大小（`-b 512`），启用 `--mlock`，开启并发槽位 `--parallel 2`。
  - 增加磁盘缓存（`diskcache`）持久化相同问题答案。
  - 使用 PyInstaller 将 FastAPI 后端打包为单个 `backend.exe`。
- **打包桌面应用**：
  - 将 `backend.exe`, `llama-server.exe`, 模型文件, Neo4j 绿色版, 启动脚本等放入 `resources` 目录。
  - 配置 `electron-builder`，生成 Windows 安装包（或免安装 exe）。
  - 编写主进程启动逻辑：启动后端子进程、Neo4j 子进程，窗口加载前等待健康检查。
- **文档与视频**：
  - 录制 3 分钟演示视频（展示智能联想、问答、步骤卡片、图谱、同步、反馈优化）。
  - 制作答辩 PPT（重点突出创新点）。

#### 成员 B
- 协助验证知识图谱规模（确保 ≥500 实体，≥80 关系）。
- 准备测试集（50 条问题及标准答案），统计准确率（Top-1 精确匹配 ≥75%）。
- 撰写知识图谱设计文档、技术报告、测试报告。
- 生成源代码调用关系图（使用 `pycallgraph` 或手动绘制）。

#### 协作与交付
- 联合测试：在 8GB 内存、4 核 CPU 的 Windows 笔记本上运行，验证响应时间 ≤3 秒，内存 ≤8GB。
- 交付物：
  - 可执行程序包（Windows 免安装）
  - 项目概要介绍、PPT、详细方案
  - 演示视频（3 分钟）
  - 知识图谱设计文档、源代码调用关系图
  - 测试报告（含准确率统计）

## 三、风险与应对

| 风险 | 应对 |
|------|------|
| llama-server 在某些 CPU 上性能不足 | 降低并发槽位为 1；使用更小模型（Phi-3-mini-3.8B） |
| PyInstaller 打包后找不到动态库 | 使用 `--collect-all llama_cpp`，并在打包后测试纯净环境 |
| 知识抽取正则覆盖率低 | 提供手工录入表单作为降级方案 |
| Neo4j 打包后路径问题 | 使用相对路径，编写 `start_neo4j.bat` 自动设置环境 |
| 反馈调参网格搜索耗时过长 | 限制搜索空间（α,β 各 3 个值），后台执行不阻塞请求 |

# NER 微调

## NER（命名实体识别）

从文本中识别出人名、地名、组织名、专业术语等有意义的实体。

```
输入："MTU4000型发动机燃油泵密封圈老化导致漏油"
NER输出：
  MTU4000型发动机 → 设备
  燃油泵          → 零件
  密封圈          → 零件
  老化            → 故障原因
  漏油            → 故障现象
```

## 微调是什么

通用 NER 模型（如 bert-base-chinese）认识"张三""北京"这种通用实体，但不认识"喷油器""叶轮"这类船舶专业术语。

微调 = 用你标注好的专业领域数据，让模型"学会"认识你的专业术语。

## 流程

```
通用 NER 模型（不认识船舶术语）
        ↓
+ 标注好的船舶故障文本（200-500 条）
        ↓
训练（几行代码，HuggingFace Transformers）
        ↓
专用 NER 模型（认识船舶术语）
```

## 标注数据长什么样

原始文本 + 每个实体的起止位置和类型：

```json
{
  "text": "发动机燃油泵密封圈老化导致漏油，需更换密封圈",
  "entities": [
    {"start": 0, "end": 3, "label": "设备"},
    {"start": 3, "end": 6, "label": "零件"},
    {"start": 6, "end": 9, "label": "零件"},
    {"start": 9, "end": 11, "label": "故障原因"},
    {"start": 13, "end": 15, "label": "故障现象"},
    {"start": 17, "end": 21, "label": "维修操作"}
  ]
}
```

## 需要多少数据？

| 数量 | 效果 | 标注耗时 |
|------|------|----------|
| 50 条 | 勉强能用，常见实体能识别 | ~1 小时 |
| 200 条 | 基本可用，大部分模式覆盖 | ~4 小时 |
| 500 条 | 比较可靠 | ~10 小时 |
| 1000 条 | 专业级 | ~20 小时 |

## 我们为什么现阶段不做？

1. 正则 + 1.5B few-shot 已经能覆盖 ~95% 场景
2. 标注 200-500 条数据太费时间（双人团队，5 周周期，时间宝贵）
3. 赛题不要求全自动抽取，手工确认是允许的
4. **保留选项**：如果实际数据中大量文本搞不定，随时可以启动微调

## 常用 NER 微调工具

| 工具 | 说明 |
|------|------|
| HuggingFace Transformers | 最主流，代码简单 |
| spaCy | 工业级 NLP 库，有中文支持 |
| label-studio | 标注工具，可视化标注界面 |
| Doccano | 另一个标注工具，更轻量 |

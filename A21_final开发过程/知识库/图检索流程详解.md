# 图检索完整流程：从用户问题到图谱答案

> 这是整个系统最核心的链路。读完这篇，你会理解自然语言是怎么变成数据库查询、再变成 LLM 能读的文本。

---

## 一、核心难题：自然语言 ≠ 数据库查询

用户不会这样提问：
```
❌ "MATCH (s:Symptom)-[r:CAUSED_BY]->(c:Cause) WHERE s.name CONTAINS '漏油' RETURN c"
```

用户会这样问：
```
✅ "发动机液压泵最近老是漏油，怎么回事"
```

**怎么把"人话"变成"数据库能执行的查询"？**

## 三、实体识别怎么做（代码示意）

```python
import jieba

# 预先准备好的船舶术语字典
ENTITY_DICT = {
    "设备": ["发动机", "电机", "接触器", "断路器", "发电机", "变压器",
             "起货机", "锚机", "舵机", "锅炉", "配电装置", "PLC",
             "三相异步电动机", "直流电机", "船用变压器", "船舶电器"],
    "零件": ["燃油泵", "密封圈", "叶轮", "线圈", "触头", "弹簧",
             "接线端子", "定子绕组", "转子", "轴承", "电刷"],
    "故障": ["漏油", "过热", "短路", "烧毁", "振动", "异响",
             "无法启动", "不能起动", "压力异常", "绝缘降低"]
}

def extract_entities(question: str) -> dict:
    """从用户问题中提取实体"""
    words = set(jieba.lcut(question))
    result = {"设备": [], "零件": [], "故障": []}
    
    for category, terms in ENTITY_DICT.items():
        for term in terms:
            if term in question:  # 字典匹配
                result[category].append(term)
    
    return result

# 示例
q = "发动机液压泵漏油怎么办"
entities = extract_entities(q)
# → {"设备": ["发动机"], "零件": ["液压泵"], "故障": ["漏油"]}
```

## 四、实体链接怎么做

提取到"发动机"后，要去 Neo4j 找到对应的节点：

```python
def link_to_graph(entities: dict, driver) -> dict:
    """把识别的实体链接到 Neo4j 中的实际节点"""
    linked = {}
    
    with driver.session() as session:
        # 找设备
        for equip in entities["设备"]:
            result = session.run("""
                MATCH (e:Equipment)
                WHERE e.name CONTAINS $name
                RETURN e.uid, e.name, e.level
                LIMIT 5
            """, name=equip)
            linked["equipment"] = [dict(r) for r in result]
        
        # 找故障现象
        for fault in entities["故障"]:
            result = session.run("""
                MATCH (s:Symptom)
                WHERE s.name CONTAINS $name
                   OR ANY(f IN s.primary_features WHERE f CONTAINS $name)
                RETURN s.uid, s.name
                LIMIT 10
            """, name=fault)
            linked["symptoms"] = [dict(r) for r in result]
    
    return linked
```

## 五、图遍历查询（核心）

有了链接到的实体，沿关系链遍历：

```python
def graph_search(linked: dict, driver) -> list[dict]:
    """核心：从 Symptom 出发，沿关系链找 Cause → Step"""
    results = []
    
    with driver.session() as session:
        for symptom in linked.get("symptoms", []):
            result = session.run("""
                // 从故障现象出发
                MATCH (s:Symptom {uid: $uid})
                
                // 找原因（按优先级）
                MATCH (s)-[r:CAUSED_BY]->(c:Cause)
                
                // 找维修步骤（可能多步）
                OPTIONAL MATCH (c)-[:FIXED_BY]->(first:Step)
                OPTIONAL MATCH (first)-[:NEXT_STEP*0..3]->(next:Step)
                
                // 找工具和注意事项
                OPTIONAL MATCH (first)-[:REQUIRES_TOOL]->(tool:Tool)
                OPTIONAL MATCH (first)-[:HAS_PRECAUTION]->(prec:Precaution)
                
                RETURN s.name AS symptom,
                       collect(DISTINCT {
                         cause: c.name,
                         priority: r.priority,
                         check_method: c.check_method,
                         check_time: c.check_time
                       }) AS causes,
                       collect(DISTINCT first.name) AS steps,
                       collect(DISTINCT tool.name) AS tools,
                       collect(DISTINCT prec.name) AS precautions,
                       first.source_doc AS source,
                       first.source_page AS page
                ORDER BY r.priority
            """, uid=symptom["uid"])
            
            for record in result:
                results.append(dict(record))
    
    return results
```

## 六、结果格式化（图结果 → LLM 能读的文本）

Neo4j 返回的是嵌套字典列表。必须转成自然语言文本，LLM 才能理解：

```python
def format_graph_results(graph_results: list[dict]) -> str:
    """把 Neo4j 结构化结果转成 LLM 能读的文本"""
    lines = ["【知识图谱检索结果】\n"]
    
    for i, item in enumerate(graph_results, 1):
        lines.append(f"故障现象: {item['symptom']}")
        lines.append(f"来源: {item['source']}，{item['page']}")
        lines.append("")
        lines.append("可能原因（按优先级排列）：")
        
        for j, cause in enumerate(item.get("causes", []), 1):
            lines.append(
                f"  {j}. {cause['cause']} "
                f"[检查方法: {cause['check_method']}, "
                f"预计耗时: {cause['check_time']}]"
            )
        
        if item.get("steps"):
            lines.append("\n维修步骤：")
            for j, step in enumerate(item["steps"], 1):
                lines.append(f"  {j}. {step}")
        
        if item.get("tools"):
            lines.append(f"\n所需工具: {', '.join(item['tools'])}")
        
        if item.get("precautions"):
            lines.append(f"\n⚠️ 注意事项: {', '.join(item['precautions'])}")
        
        lines.append("")
    
    return "\n".join(lines)
```

## 七、最终注入 Prompt

```python
def build_prompt(question: str, graph_text: str, vector_docs: list, bm25_docs: list):
    """三路结果融合后构建最终 Prompt"""
    
    # 所有检索结果编号
    all_chunks = []
    for doc in vector_docs + bm25_docs:
        all_chunks.append(f"[{len(all_chunks)+1}] 来源: {doc['source']}, {doc['page']}\n{doc['text']}\n")
    
    prompt = f"""你是船舶电气设备维修专家。

请根据以下资料回答问题。如果资料中没有相关信息，直接回答"未找到相关信息"。

{graph_text}

参考资料：
{''.join(all_chunks)}

用户问题：{question}

要求：
1. 列出可能的故障原因，按概率从高到低排列
2. 每个原因标注检查方法和预计耗时
3. 列出维修步骤
4. 涉及的引用必须标注编号，如 [1][2]
5. 安全注意事项单独列出
"""
    return prompt
```

## 八、整个流程的时间分布

```
用户输入
  │ 0.00s
  ├─ jieba 分词 + 字典 NER              0.02s
  ├─ Neo4j 实体链接（模糊搜索）           0.05s
  ├─ Neo4j 图遍历查询                    0.10s
  ├─ 向量检索 (embedding + Chroma)       0.25s
  ├─ BM25 检索                           0.05s
  ├─ RRF 融合 + 去重                      0.02s
  ├─ Prompt 构建                          0.01s
  │ 0.50s ← 检索阶段总计
  ├─ llama.cpp prompt 处理（缓存后~250tk）  1.5s
  ├─ 首 token 生成                        0.05s
  │ 2.05s ← 用户看到第一个字 ✅
  └─ 后续 token 流式输出...
```

## 九、关键理解

1. **图检索不是替代向量检索，是互补**。图检索给出结构化因果链，向量检索给出相关文档段落。
2. **实体链接是最容易被忽略的难点**。用户说的"发动机"在库里可能叫"船用电机"——模糊匹配和同义词映射是必须的。
3. **Cypher 是沿着关系链遍历**，不是全文搜索。威力在于"漏油 → 什么原因 → 怎么修 → 注意什么"一次查询全出来。
4. **LLM 不直接读 Neo4j**。我们把图结果转成文本，和向量/BM25 结果一起注入 Prompt，LLM 像"读资料"一样阅读。

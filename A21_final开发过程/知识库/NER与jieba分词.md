# NER 与 jieba 分词

## NER（命名实体识别）

### 一句话解释

从一句话里**识别出"人名、地名、专业名词"等有意义的实体**。

### 例子

```
"MTU4000型发动机的B12号燃油泵压力异常，需要更换密封圈"

NER 识别结果：
  设备：MTU4000型发动机
  零件：B12号燃油泵
  故障：压力异常
  维修：更换密封圈
```

### 在我们项目中的作用

用户输入"发动机液压泵漏油怎么办"——

NER 提取：设备=发动机, 零件=液压泵, 故障=漏油

有了这些实体，才能：
- 去 Neo4j 里精确查询 "液压泵" 的故障关系
- 去向量库里搜 "液压泵 漏油" 而不是搜整句话

### 实现方式

| 方案 | 说明 | 适合场景 |
|------|------|----------|
| 字典匹配 | 把所有已知零件名、故障名做成字典，直接匹配 | 领域封闭、实体有限 |
| jieba 分词 + 词性标注 | 用 jieba 分出词并标注词性（名词、动词等） | 通用中文 |
| 微调小模型 | 训练一个专门的 NER 模型 | 领域开放、需要泛化 |

建议：先用**字典匹配 + jieba**（简单可靠），不够用再加小模型。

## jieba 分词

### 一句话解释

把中文句子**切成有意义的词**。中文不像英文有空格，需要专门的分词工具。

### 例子

```python
import jieba

jieba.lcut("发动机液压泵漏油怎么办")
# → ['发动机', '液压泵', '漏油', '怎么办']
```

### 为什么需要分词？

- 检索时不能搜"液压泵漏油怎么"这种破碎的词
- 实体识别需要知道词的边界
- BM25 需要以词为单位统计频率

### jieba 的三种模式

| 模式 | 例子 | 适用 |
|------|------|------|
| 精确模式 | 发动机/液压泵/漏油 | 检索、NER |
| 全模式 | 发动/动机/发动机/液压/液压泵/漏油 | 冗余覆盖 |
| 搜索引擎模式 | 发动机/动机/液压泵/压泵/漏油 | 搜索建议 |

# 实体识别与实体链接

## 一句话解释

实体识别 = 从句子中找出"谁是什么"。实体链接 = 把这个"谁"对应到数据库里的具体记录。

## 类比

```
用户在搜索引擎里输入："北京天气"

实体识别：找出"北京"是一个地名
实体链接：把"北京"对应到数据库里 id=110000 的那条城市记录

没有实体链接 → 搜索"北京"可能返回"南京"（因为都带"京"）
有实体链接   → 精确查到北京市的天气数据
```

## 在我们的项目里

```
用户问题："发动机液压泵漏油怎么办"

第一步：实体识别 (NER)
  → 设备: "发动机"
  → 零件: "液压泵"
  → 故障: "漏油"

第二步：实体链接 (Entity Linking)
  → "发动机" → Neo4j 里有吗？
     - 直接搜"发动机" → 没有
     - 模糊搜 CONTAINS '发动机' → 找到"船用电机"、"船舶电器"等候选
     - 选最相关的：E_MOTOR_CAT (船用电机)
  
  → "液压泵" → Neo4j 里有吗？
     - 搜"液压泵" → 当前数据里可能没有
     - → 链接失败，放弃图检索这条路
     - → 但向量检索和 BM25 仍然可以搜到相关文档

  → "漏油" → Neo4j 里有吗？
     - 搜 Symptom 的 name 和 primary_features
     - 找到"油封泄漏"、"管路漏油"等 → 链接成功
```

## 为什么实体链接容易失败？

用户用的词 ≠ 数据库里的词：

| 用户说 | 数据库里叫 |
|--------|-----------|
| 发动机 | 船用电机 |
| 打不着火 | 不能起动 |
| 冒黑烟 | 排气异常 |
| 嗡嗡响 | 异常振动 |

**解决方案**：同义词映射表 + 模糊匹配。

```python
# 同义词表
SYNONYM_MAP = {
    "发动机": ["船用电机", "柴油机", "引擎"],
    "打不着火": ["不能起动", "无法启动", "启动失败"],
    "漏油": ["油封泄漏", "管路漏油", "渗油", "漏液"],
}

def expand_query(entity_name: str) -> list[str]:
    """把用户词扩展为可能的数据库词"""
    candidates = [entity_name]
    for user_word, db_words in SYNONYM_MAP.items():
        if user_word in entity_name:
            candidates.extend(db_words)
    return candidates
```

## 实体链接失败怎么办？

**不要慌，这是正常的。** 图检索只是三路之一：

```
实体链接成功 → 图检索拿到结构化因果链 ✅
实体链接失败 → 图检索返回空结果
              向量检索 + BM25 仍然正常工作 ✅
              LLM 也有足够的信息回答问题
```

## 和纯向量检索的区别

| | 纯向量检索 | 加了实体链接的图检索 |
|------|-----------|-------------------|
| "漏油原因" | 返回含有"漏油"的文档段落 | 返回"密封圈老化→更换密封圈→注意事项"完整因果链 |
| "三相异步电动机不能起动" | 返回相关文档 | 精确返回3个原因（按优先级）+ 维修步骤 + 工具 |
| "这个零件发热怎么回事" | 不知道"这个零件"是哪个 | 如果上轮对话提到了"接触器"，这次就能精确查 |

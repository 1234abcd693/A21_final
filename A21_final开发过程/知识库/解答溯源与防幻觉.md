# 解答溯源与防幻觉

## 一句话解释

溯源 = 让用户能验证 AI 说的每一个论断来自哪里。防幻觉 = 在用户看到之前，系统自己先验证。

## 为什么需要？

大模型（即使是 1.5B 小模型）会"编造"内容。这个问题叫"幻觉"。RAG 让模型根据资料回答，但模型仍可能在资料之外添油加醋。

溯源让用户有能力验证；防幻觉让系统在有把握时才给用户看。

## 两层防护

```
AI 生成完成
    │
    ├── 第一层：引用完整性检查
    │   答案里有 [1] [2] [3]，但检索只返回了 2 条文档？
    │   → [3] 是编的 → 标记 🟡
    │
    ├── 第二层：事实一致性检查
    │   答案里说"需要更换燃油泵"
    │   用 jieba 提取: ["更换", "燃油泵"]
    │   在源文档中搜"燃油泵"→ 找到了 ✅
    │   在源文档中搜"更换"→ 找到了 ✅
    │   → 全部通过 → 🟢
    │
    └── 综合判定
        两层都过 → 🟢 可信
        一层失败 → 🟡 部分可信 (显示警告)
        两层都失败 → 🔴 不可信 (红色横幅)
```

## 溯源链

每条回答记录它依赖了哪些文档和哪些图谱节点：

```
回答中某句话 → [1] 维护手册 P42
                    ↓
              文档片段 "接触器线圈烧毁时应先测量绝缘电阻..."
                    ↓
              图谱节点: S_CONT_02 (线圈烧毁)
                    ↓        → C_VOLTAGE_LOW (电压过低)
                    ↓        → ST_COIL_REPLACE (更换线圈)
            用户可以点击任何一个 → 跳转验证
```

## 用户能做什么

| 操作 | 行为 |
|------|------|
| 点击 [1] | 弹出源文档原文 |
| 点击溯源链中的节点名 | 图谱跳转 + 该节点闪烁 |
| 点击"展开完整路径" | 图谱中高亮 Symptom→Cause→Step 完整链 |

## 为什么不是"拒绝回答"

有些系统在验证不通过时直接拒绝回答。我们不这样做，因为：
- 部分可信（🟡）仍然可能对用户有帮助
- 拒绝回答会让系统显得"不好用"
- 只要明确标注可信度，用户自己会判断

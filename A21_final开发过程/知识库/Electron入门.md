# Electron 入门

## 一句话解释

Electron = 用写网页的技术（HTML + CSS + JavaScript）**做出一个真正的 Windows 桌面程序**。

## 类比

你平时用的 VS Code、Discord、飞书、Obsidian —— 这些全都是 Electron 做的。它们看起来是桌面软件，骨子里是网页。

## 为什么选 Electron？

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Electron + Vue** | 桌面应用体验好、窗口管理、系统托盘、文件对话框 | 开发工作量较大 |
| Streamlit | Python 就能写 UI，极其简单 | 浏览器打开，不是桌面应用、UI 自由度低 |
| Flask + HTML | 轻量、Python 全栈 | 还是浏览器打开、UI 效果一般 |

对于"Windows 平台桌面应用程序"这个目标，Electron 是正确的选择。

## Electron 的核心结构

```
┌──────────────────────────────────────────┐
│              Electron 应用                 │
│                                           │
│  ┌─────────────────────────────────┐     │
│  │  Main Process (主进程)           │     │
│  │  · Node.js 环境                  │     │
│  │  · 创建窗口、管理应用生命周期       │     │
│  │  · 启动/停止 Python 后端子进程     │     │
│  │  · 读写本地文件、系统对话框        │     │
│  └──────────────┬──────────────────┘     │
│                 │ IPC 通信                │
│  ┌──────────────┴──────────────────┐     │
│  │  Renderer Process (渲染进程)      │     │
│  │  · Vue 3 写的界面                │     │
│  │  · 用户看到和操作的窗口内容        │     │
│  │  · 通过 HTTP 和 Python 后端通信   │     │
│  │  · 通过 IPC 让主进程做系统操作     │     │
│  └─────────────────────────────────┘     │
│                                           │
│  ┌─────────────────────────────────┐     │
│  │  Python 后端 (FastAPI)            │     │
│  │  · 被主进程启动的子进程            │     │
│  │  · 跑在 localhost:8000            │     │
│  │  · Vue 通过 HTTP 调用它           │     │
│  └─────────────────────────────────┘     │
└──────────────────────────────────────────┘
```

## 两个进程为什么分开？

| | 主进程 (Main) | 渲染进程 (Renderer) |
|------|--------------|-------------------|
| 运行环境 | Node.js | 浏览器环境（Chromium） |
| 能做什么 | 系统操作、文件读写、启动子进程 | 显示界面、发起 HTTP 请求 |
| 不能做什么 | 直接操作 DOM（没有界面） | 直接操作文件系统 |
| 类比 | 后台管理员 | 前台服务员 |

**安全原因**：不能让网页代码随便读写用户文件。所以渲染进程如果需要文件操作，必须通过 IPC 告诉主进程，主进程代劳。

## 和普通网页的区别

| | 普通网页 | Electron 应用 |
|------|---------|-------------|
| 运行在哪 | 浏览器 | 自己的 Chromium 窗口 |
| 有地址栏吗 | 有 | 没有 |
| 能操作本地文件吗 | 不能（安全限制） | 能（通过主进程） |
| 能启动其他程序吗 | 不能 | 能 |
| 需要服务器吗 | 需要部署在服务器上 | 不需要，一切在本地 |

## 我们的 Electron 架构

```
用户双击 A21.exe
    │
    ▼
Electron 主进程启动
    │
    ├── 启动 Python 后端子进程（FastAPI，端口 8000）
    ├── 启动 Neo4j 子进程（如果没运行）
    ├── 启动 llama-server 子进程（端口 8080）
    │
    ├── 等待所有服务就绪（轮询 health check）
    │
    ├── 创建窗口，加载 Vue 前端
    │
    └── Vue 前端通过 HTTP 调用 Python 后端 API
```

用户关闭窗口时，主进程负责依次关闭所有子进程。

## 你需要关心的

你主要负责 Vue 前端代码。Electron 壳子部分（`main.js`、`preload.js`）写好后基本不需要动。

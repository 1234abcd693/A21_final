# 语音识别（Whisper.cpp）

## 一句话解释

whisper.cpp = 语音版的 llama.cpp。把音频文件转成文字，完全离线运行在 CPU 上。

## 和 llama.cpp 的关系

| | llama.cpp | whisper.cpp |
|------|-----------|------------|
| 做什么 | 文字 → 文字（问答） | 声音 → 文字（转录） |
| 作者 | 同一个人 (ggerganov) | 同一个人 |
| 运行方式 | `llama-server.exe` 启动 HTTP 服务 | `whisper-server.exe` 启动 HTTP 服务 |
| 文件格式 | `.gguf` | `.ggml` / `.bin` |
| 离线 | ✅ | ✅ |
| CPU 运行 | ✅ AVX2 加速 | ✅ AVX2 加速 |

## 我们为什么用它？

1. **完全离线** — 和 llama.cpp 一样，不需要网络
2. **同一个生态** — 启动方式、打包方式、架构完全一致
3. **Electron 完美支持** — 浏览器自带录音 API
4. **轻量** — base 模型 140MB，毫秒级转录

## 模型大小选择

| 模型 | 大小 | 中文效果 | CPU 速度 (5秒音频) |
|------|------|:--:|------|
| tiny | 75 MB | ⭐⭐ | ~0.3s |
| base | 140 MB | ⭐⭐⭐ | ~0.5s |
| small | 460 MB | ⭐⭐⭐⭐ | ~1.2s |
| medium | 1.5 GB | ⭐⭐⭐⭐⭐ | ~3s |

**推荐 base**：140MB、够快、中文基本够用。如果准确率不满意，升级到 small（+320MB）。

## 工作流程

```
用户点击麦克风按钮 🎤
    │
    ▼
前端 MediaRecorder API 开始录音
    │  浏览器原生能力，不需要额外库
    ▼
用户说话："发动机液压泵漏油怎么办"
    │
    ▼
松开按钮 → 录音结束 → WAV 音频 blob
    │
    ▼
POST /api/v1/transcribe  (发送音频)
    │
    ▼
FastAPI 后端 → 转发给 whisper-server (localhost:8081)
    │
    ▼
whisper.cpp 转录 → 返回文本 "发动机液压泵漏油怎么办"
    │
    ▼
文本填入输入框 → 用户确认/修改 → 点发送
```

## 为什么不是实时转录而是按需？

实时语音转文字（边说边出字）需要持续占用 CPU，对 8GB 内存的普通 PC 压力大。按需录音（按住说话，松手转录）简单可靠，2 秒内出结果。

## 启动命令

```powershell
.\whisper-server.exe `
  -m ..\model\ggml-base.bin `
  --host 127.0.0.1 `
  --port 8081 `
  -l zh                    # 中文
```

FastAPI 后端新增路由：

```python
@app.post("/api/v1/transcribe")
async def transcribe(audio: UploadFile):
    # 接收前端发来的音频文件
    audio_bytes = await audio.read()
    # 转发给 whisper-server
    response = httpx.post(
        "http://127.0.0.1:8081/inference",
        files={"file": ("audio.wav", audio_bytes)}
    )
    return response.json()  # {"text": "发动机液压泵漏油怎么办"}
```

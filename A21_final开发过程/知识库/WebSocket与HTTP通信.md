# WebSocket vs HTTP 通信

## WebSocket

**持续连接**，客户端和服务器建立连接后保持打开，双方可以随时发送数据。

```
客户端 ←→ 服务器（连接保持）
  → 发送数据块1
  ← 返回部分结果
  → 发送数据块2
  ← 返回最终结果
```

适合：实时聊天、在线游戏、**持续流式语音识别**

## HTTP

**一次请求一次响应**，客户端发送请求，服务器处理完返回结果，连接关闭。

```
客户端 → 发送完整请求 → 服务器处理 → 返回结果 → 连接关闭
```

适合：网页浏览、API 调用、**录制后批量语音识别**

## 我们为什么从 WebSocket 换到 HTTP

WebSocket 流式语音需要 `AudioContext` + `ScriptProcessor`，在不同浏览器/Electron 中兼容性差，容易断开。

换成 HTTP 模式：录制完整音频 → 一次性发送 → 等待结果。简单可靠，2-3 秒内出结果。

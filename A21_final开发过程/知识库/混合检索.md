# 混合检索（Hybrid Search）

## 一句话解释

同时用**多种检索方式**查找资料，然后**融合排序**，取各家之长。

## 为什么单一检索不够？

### 场景 1：用户问"发动机打不着火怎么办"

| 检索方式 | 结果 |
|----------|------|
| 关键词（BM25） | 没搜到"打不着火"这个精确词组 → 漏了 |
| 向量检索 | 理解"打不着火"≈"无法启动" → 找到了！ |
| 图检索 | 知道"发动机"和"启动系统"有关 → 找到了相关零件 |

### 场景 2：用户问"燃油压力异常 3.2MPa"

| 检索方式 | 结果 |
|----------|------|
| 关键词（BM25） | 精确匹配"燃油压力"+"3.2MPa" → 精准命中 |
| 向量检索 | 3.2 和 3.1 在向量空间里很近，但语义上并不相似 → 误召回 |
| 图检索 | "燃油压力"→"燃油泵"→"压力调节阀" → 找到了因果链 |

## 三路检索对比

| | BM25（关键词） | 向量检索 | 图检索 |
|------|-----------|--------|------|
| 原理 | 统计词频，精确匹配 | 语义向量距离，模糊匹配 | 实体关系遍历，结构化查询 |
| 擅长 | 术语、编号、精确数值 | 同义词、近义表达、口语化问题 | 因果链、层级关系、关联推理 |
| 盲区 | 不认同义词 | 对数字不敏感 | 无法检索非结构化描述 |
| 工具 | rank_bm25 库 | Chroma + embedding 模型 | Neo4j + Cypher 查询 |

## 融合策略

```
用户问题
    │
    ├─→ 实体识别 → Neo4j Cypher 查询 → 图结果（三元组列表）
    ├─→ 向量化   → Chroma 相似度搜索 → 向量结果（文档片段+分数）
    ├─→ 关键词   → BM25 检索         → BM25结果（文档片段+分数）
    │
    └─→ 融合层
         │
         方案A：RRF（Reciprocal Rank Fusion）
           · 不关心分数绝对值，只看排名
           · score = Σ 1/(k + rank)  对三路各自的排名求和
           · 优点：不需要调权重，开箱即用
         
         方案B：加权融合
           · score = α·BM25 + β·Vector + γ·Graph
           · 需要手动调 α, β, γ
           · 优点：可针对场景优化，反馈闭环可以自动调参
    │
    └─→ 重排序（Reranker）
         · 对融合后的 Top-10 做精细排序
         · 选出最相关的 Top-5 注入 Prompt
```

## 我们选什么？

建议先上 RRF（简单可靠），等反馈闭环跑起来后，切换到加权融合并自动调参。

# BM25 关键词检索

## 一句话解释

传统的**搜索引擎算法**，根据关键词出现的频率和稀有度来打分排序。

## 类比

Google 搜索的核心思想（简化版）：
- 搜索词在文档中出现次数多 → 分数高
- 搜索词在所有文档中都很常见（如"的""了"） → 不重要，扣分
- 文档越长，需要出现次数更多才能得高分 → 长度归一化

## 和向量检索的对比

| | BM25 | 向量检索 |
|------|------|---------|
| 匹配方式 | 精确词匹配 | 语义相似 |
| "打不着火" vs "无法启动" | ❌ 词不同，不匹配 | ✅ 语义相近 |
| "螺栓扭矩 120N·m" | ✅ 精确匹配数字 | ⚠️ 数字敏感度低 |
| 速度 | 极快，毫秒级 | 需计算 embedding，稍慢 |
| 依赖 | 无，纯统计 | 需要 embedding 模型 |

## 在我们项目中的位置

BM25 专门负责**精确术语匹配**：
- 型号、编号："燃油泵型号 XYZ-123"
- 数值："压力 3.2MPa"
- 专业缩写："MTU 4000 系列"

Python 实现用 `rank_bm25` 库，几行代码即可。

## 为什么 BM25 不能单独用？

用户说的"打不着火"和手册里的"无法启动"是两个完全不同的词，BM25 匹配不到。必须配合向量检索覆盖语义相似的情况。

# A21 前端交互设计 v2.0

> 版本: v2.0 | 核心：图谱为交互中枢，关键词检索融入图谱，完整解答溯源方案

---

## 一、设计理念

**知识图谱是核心交互界面，不是附件。** 搜索、提问、溯源都围绕图谱展开：

```
旧: 智能问答 | 关键词检索 | 图谱可视化  ← 三个独立页面
新: 图谱搜索 + 详情面板 + 问答面板        ← 图谱是中枢
```

## 三、关键词检索 = 图谱搜索

**不再有独立的关键词检索页面。搜索直接在图谱上发生。**

```
输入 "接触器 线圈" → 实体链接匹配 Neo4j
  → 返回: 线圈烧毁(0.95) / 线圈过热(0.82) / 短路(0.78)
  → 匹配节点高亮+脉冲 / 非匹配节点半透明 / 图谱平移居中
```

| 层次 | 用户看到的 | 用户能做的 |
|------|-----------|-----------|
| 图谱 | 匹配节点高亮，非匹配节点半透明 | 拖拽缩放点击展开 |
| 右侧 | 匹配节点列表（按分数排） | 点击跳转 |
| 底部 | 节点旁的 💬 → 自动提问 | 编辑发送 |

## 四、解答溯源方案

赛题要求："确保系统输出的维修建议可追溯、有依据，避免误导性内容。"

### 4.1 溯源卡片

```
┌── 溯源链 ──────────────────────────┐
│ 引用 [1]                           │
│  文档: 《船舶电气设备维护与修理》     │  ← 点击展开原文
│  页码: 第四章第二节 P42             │
│  图谱路径:                          │
│    🔴 线圈烧毁 → 🟠 电压过低        │  ← 点击跳转到图中节点
│    🟠 电压过低 → 🟢 更换线圈        │
│ 🟢 可信度: 全部已验证 (3/3引用)     │
│          (5/5关键词在文档中出现)      │
└────────────────────────────────────┘
```

### 4.2 可信度三级

| 等级 | 条件 | 显示 |
|:--:|------|------|
| 🟢 可信 | 引用全在范围内 + 关键词全可溯源 | "全部引用已验证" |
| 🟡 部分可信 | 部分引用/关键词无法溯源 | "⚠️ 部分信息未经确认" |
| 🔴 不可信 | 无引用 + 验证全部失败 | 红色横幅："此回答无法验证" |

### 4.3 防幻觉两层验证

| 层 | 验证内容 | 失败 |
|------|---------|------|
| 引用完整性 | 答案中 [N] 在检索范围内？ | 🟡 |
| 事实一致性 | 答案关键名词在源文档中出现？ | 🟡 |
| **综合** | 两层都过→🟢 / 一层失败→🟡 / 两层都失败→🔴 | |

### 4.4 节点颜色编码

| 颜色 | 实体 | 示例 |
|:--:|------|------|
| 🔴 | Symptom | 线圈烧毁 |
| 🟠 | Cause | 电压过低 |
| 🟢 | Step | 更换线圈 |
| 🔵 | Equipment | 船用电机 |
| 🟣 | Precaution | 断电操作 |

## 六、核心交互流程

```
图谱探索→问答:  点击节点→详情→💬→AI流式→溯源卡片→点击溯源节点→图谱跳转
搜索→定位→问答:  搜索框输入→高亮→右侧列表→跳转→💬→同上
直接提问→溯源:   底部输入→AI回答→展开溯源卡片→点击节点→图谱跳转
```

## 二、两种输入模式

### 模式 A：智能问答（默认）

用户输入自然语言问题，系统走完整 RAG 管道。

**行为**：
- 输入框占 2-3 行，支持多行
- 回车发送，Shift+回车换行
- 回答流式逐字展示（SSE）
- 引用 [1] 可点击/悬浮查看来源
- 每条回答下方 👍👎📋
- 支持多轮对话（上下文保持）

**适用场景**：船员不知道确切原因，描述现象寻求诊断。

### 模式 B：关键词检索

用户输入关键词（空格分隔），系统加重 BM25 权重进行精确匹配。

**行为**：
- 输入框单行（更像搜索框）
- 回车直接搜索
- 结果以**列表**展示（非对话气泡）
- 每条结果展示：标题、摘要、来源文档+页码
- 点击某条结果 → 跳转到智能问答模式，以该文档内容为基础提问

**适用场景**：维修工程师知道术语，想快速查特定内容。

### 两种模式的关系

```
智能问答模式（用户不知道怎么描述）
    ↓ 用户看到引用 [1]，想深入了解
    ↓ 点击引用 → 跳转到关键词检索，自动填好搜索词
    ↓
关键词检索模式（用户知道要查什么）
    ↓ 用户找到感兴趣的文档
    ↓ 点击"以此文档问答" → 跳转到智能问答，自动引用该文档
    ↓
回到智能问答模式
```

**两种模式不是孤立的两页，而是可互相跳转的。**

## 四、各组件交互说明

### 4.1 Sidebar.vue

| 导航项 | 点击行为 |
|--------|----------|
| 知识库浏览 | 主区域切换为设备分类树 + 故障列表（浏览模式） |
| 历史对话 | 主区域显示历史对话列表，点击可恢复 |
| 知识图谱 | 弹出 GraphModal，全屏展示力导向图 |
| 文档上传 | 弹出 UploadModal，拖拽上传 + 图谱预览 |
| 数据导出 | 弹出 SyncModal (导出模式) |
| 数据导入 | 弹出 SyncModal (导入模式) |
| 参数优化 | 触发后端优化，显示进度，完成后提示 |

### 4.2 ChatView.vue（智能问答）

```
1. 用户输入问题 → 点发送
2. 输入框清空，用户消息气泡出现在列表底部
3. AI 正在思考... (加载动画)
4. 首 token 到达 → 替换加载动画，开始流式显示
5. 回答完成 → 显示 👍👎📋 按钮
6. 用户可继续输入追问（多轮上下文）
```

**状态管理**：
```javascript
// 当前对话
const messages = ref([...])  // 消息列表
const isStreaming = ref(false)  // 是否正在流式接收

// 每条消息结构
{
  id: 'msg_001',
  role: 'user' | 'assistant',
  content: '...',
  citations: [{num: 1, doc: '...', page: '...'}],
  timestamp: '...'
}
```

### 4.3 SearchView.vue（关键词检索）

```
1. 用户输入关键词 → 回车
2. 搜索框上方出现"搜索中..."动画
3. 结果列表渲染（每条结果: 标题 + 摘要 + 来源）
4. 用户可：
   - 点击某条结果 → "以此文档问答" → 切换到智能问答模式
   - 修改关键词 → 重新搜索
```

**状态管理**：
```javascript
const keyword = ref('')
const results = ref([])
const isSearching = ref(false)

// 每条结果结构
{
  id: 'doc_001',
  title: '接触器线圈烧毁的原因与维修',
  snippet: '接触器线圈内部短路导致...',
  source_doc: '船舶电气设备维护与修理',
  source_page: '第四章第二节',
  score: 0.89
}
```

### 4.4 InputBox.vue（智能问答输入框）

| 交互 | 行为 |
|------|------|
| 输入文字 | 实时显示，自动联想（调 /symptoms 接口） |
| 回车 | 发送消息 |
| Shift+回车 | 换行 |
| 点发送按钮 | 发送消息 |
| 输入时下拉联想 | 列表显示已有的故障现象，点击直接填入 |

### 4.5 SearchBar.vue（关键词检索框）

| 交互 | 行为 |
|------|------|
| 输入关键词 | 空格分隔多个词 |
| 回车 | 触发搜索 |
| 右侧搜索图标 | 触发搜索 |

### 4.6 MessageBubble.vue（消息气泡）

| 元素 | 行为 |
|------|------|
| 用户消息 | 右对齐，蓝色气泡，显示时间 |
| AI 消息 | 左对齐，白色气泡，流式逐字显示 |
| 引用 [N] | 蓝色可点击，悬浮显示来源文档+页码 |
| 低可信警告 ⚠️ | 黄色横幅，在回答末尾 |
| 👍 按钮 | 点击变绿，发送 rating=1 |
| 👎 按钮 | 点击变红，发送 rating=-1 |
| 📋 按钮 | 复制回答到剪贴板 |

### 4.7 FeedbackButtons.vue

| 状态 | 显示 |
|------|------|
| 未投票 | 👍👎 都是灰色 |
| 点赞后 | 👍 绿色实心，👎 灰色 |
| 点踩后 | 👎 红色实心，👍 灰色 |
| 已投票 | 再次点击取消投票 |

### 4.8 GraphModal.vue（知识图谱可视化）

- D3.js 力导向图，全屏弹窗
- 节点=实体（不同颜色区分类型），连线=关系（带箭头和标签）
- 拖拽移动节点，滚轮缩放
- 点击节点：展开相邻节点 + 右侧显示属性面板
- 右下角图例：Symptom(红) / Cause(橙) / Step(绿) / Equipment(蓝) / ...

## 五、模式切换与语音输入

### 模式切换栏

```
┌──────────────────────────────────────────────┐
│  [💬 智能问答]  [🔍 关键词检索]      [🎤 语音] │
└──────────────────────────────────────────────┘
```

- 点击 💬/🔍 切换主区域视图
- 当前模式高亮（蓝色底）
- 切换模式时不清空已有内容

### 语音输入 🎤

```
🎤 按钮
  ├── 按下并按住 → 开始录音（按钮变红 + "正在聆听..."）
  ├── 松开 → 停止录音 → 发送到 /transcribe
  ├── 识别结果 → 填入输入框（不自动发送）
  └── 用户可编辑识别结果 → 手动点发送
```

## 六、自动联想

输入时（智能问答模式），每输入 2 个字触发一次联想：

```
用户输入: "接触器线"
    │
    ▼
GET /api/v1/symptoms?keyword=接触器线
    │
    ▼
返回: ["接触器线圈烧毁", "接触器线圈过热", "接触器线间短路"]
    │
    ▼
下拉列表显示，用户点击或 ↓↑ 选择，回车确认
```

联想框在输入框正下方，最多显示 5 条，点击后填入输入框。

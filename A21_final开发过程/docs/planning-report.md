## 二、已确认的技术决策

### 2.1 系统架构（6 层）

```
┌──────────────────────────────────────────────┐
│ 1. 前端交互层                                 │
│    聊天界面(SSE流式) / 图谱可视化 / 文档上传    │
│    反馈按钮(👍👎) / 数据同步入口               │
├──────────────────────────────────────────────┤
│ 2. 服务层 (FastAPI)                          │
│    路由分发 / 请求验证 / SSE 流式推送          │
├──────────┬──────────────┬────────────────────┤
│ 3. RAG层 │ 4. 反馈层     │ 5. 数据层           │
│ 检索模块 │ 收集/存储     │ 知识图谱(Neo4j)     │
│ 生成模块 │ 参数优化      │ 向量库(Chroma)      │
│ 答案验证 │              │ 对话历史/上下文      │
├──────────┴──────────────┴────────────────────┤
│ 6. 工具层                                     │
│    知识抽取 / 文档解析 / 数据清洗               │
│    U盘同步(导入导出) / 报告生成                 │
└──────────────────────────────────────────────┘
```

### 2.2 模型选型

| 项 | 选择 | 理由 |
|----|------|------|
| 模型 | **Qwen2.5-1.5B-Instruct** | ≤7B 限制下唯一满足 3s TTFT + 8GB 内存约束 |
| 量化 | Q4_K_M（~1 GB） | 最佳速度/质量平衡，普通 CPU 可流畅运行 |
| 推理引擎 | **llama.cpp**（llama-server 模式） | C++ 原生性能，AVX2 加速，支持 prompt 缓存 |
| 优化 | AVX2 + 4线程 + mlock + flash-attn | 最大化 CPU 推理速度 |

### 2.3 检索策略（三路融合）

```
用户问题 → jieba分词 → NER实体抽取
    │
    ├── 实体名 → Neo4j Cypher → 图检索结果（结构化三元组）
    ├── 关键词 → BM25 检索    → BM25 结果（精确匹配片段）
    └── Embedding → Chroma   → 向量结果（语义相似片段）
    │
    └── RRF 融合 → Reranker 重排序 → Top-5 注入 Prompt
```

| 检索方式 | 工具 | 职责 |
|----------|------|------|
| 图检索 | Neo4j + Cypher | 实体关系推理、因果链 |
| 向量检索 | Chroma + MiniLM | 语义相似、同义词覆盖 |
| BM25 | rank_bm25 | 精确术语/数值匹配 |
| 融合 | RRF（先）→ 加权（后，反馈驱动） | 三路结果统一排序 |
| 重排序 | Cross-encoder Reranker | 精细排序，提升 Top-5 精度 |

### 2.4 NLP 技术栈

| 环节 | 技术 | 说明 |
|------|------|------|
| 分词 | jieba | 中文分词基础 |
| 实体识别 | 字典匹配 + jieba 词性标注 | 提取设备名、零件名、故障类型 |
| Query 改写 | 同义词扩展 / LLM 辅助 | "打不着火"→"无法启动" |
| 文本分块 | 滑动窗口 256-512 token + 10-20% 重叠 | 平衡检索精度和上下文完整性 |
| Embedding | paraphrase-multilingual-MiniLM-L12-v2 | 384 维，中英双语，轻量 |
| 答案验证 | jieba 关键词 + 源文档匹配 | 防幻觉，低可信警告 |

### 2.5 防幻觉机制

```
生成答案 → 提取关键名词(jieba) → 检查是否在检索片段中出现
                                    ├─ 全部出现 → 正常输出
                                    └─ 有未出现 → 标记 "⚠️ 部分信息未经源文档确认"
```

### 2.6 Prompt 优化

- **Prompt 缓存**：系统指令永久缓存，每次只处理新内容（节省 ~50% TTFT）
- **结构化输出**：固定格式（原因列表 + 维修步骤 + 引用编号 + 注意事项）
- **上下文窗口**：≤ 2048 token，确保内存和速度

## 五、性能约束与应对

| 约束 | 应对措施 |
|------|----------|
| ≤3s 响应 | 1.5B 模型 + Prompt 缓存 + 压缩上下文 + AVX2 优化 |
| ≤8GB 内存 | Q4_K_M 量化(~1GB) + KV cache 8bit + 精简依赖 |
| 完全离线 | llama.cpp + 本地 Neo4j + 本地 Chroma + 无网络依赖 |
| ≥500 实体 | 赛题数据 + 知识抽取补充 + 手工录入 |
| ≥75% Top1 准确率 | 三路检索融合 + Reranker + 结构化 Prompt + 答案验证 |

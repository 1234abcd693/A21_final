## 二、各层职责与接口

### 2.1 前端交互层

| 职责 | 说明 |
|------|------|
| 聊天界面 | 输入框 + 流式气泡 + 引用悬浮（[1] 显示来源） |
| 图谱可视化 | D3.js / vis.js 渲染力导向图，点击节点展开 |
| 反馈交互 | 每条回答下方 👍/👎 按钮，可选文字反馈 |
| 管理界面 | 文档上传、数据同步、参数面板 |

### 2.2 服务层

| 路由 | 方法 | 说明 |
|------|------|------|
| `/api/v1/ask` | POST | 问答（SSE 流式返回） |
| `/api/v1/symptoms` | GET | 故障现象联想（输入框自动补全） |
| `/api/v1/parse` | POST | 上传文档，返回抽取候选 |
| `/api/v1/confirm` | POST | 确认导入候选到 KG |
| `/api/v1/feedback` | POST | 提交 👍/👎 |
| `/api/v1/export` | GET | 导出数据包（zip） |
| `/api/v1/import` | POST | 导入数据包 |
| `/api/v1/history` | GET | 历史对话列表 |
| `/api/v1/report` | GET | 导出问答报告 |

### 2.3 RAG 层

**检索模块**（三路融合）：
```
用户问题 → jieba分词 → 实体抽取(NER/字典)
    │
    ├─→ 实体名 → Neo4j Cypher ─→ 图结果（三元组）
    ├─→ 关键词 → BM25 检索   ─→ BM25 结果（文档片段）
    └─→ Embedding → Chroma   ─→ 向量结果（文档片段）
    │
    └─→ RRF 融合排序 → Reranker 重排序 → Top-5 注入 Prompt
```

> 三路互补：图检索擅长实体关系推理，BM25 擅长精确术语匹配，向量检索擅长语义相似。

**生成模块**：
```
Top-K 片段 + 用户问题 + 对话历史 → Prompt 模板 → llama-server → SSE 流式输出

生成完毕后 → 答案验证：
  · 引用完整性检查（[N] 是否在检索范围内）
  · 事实一致性检查（答案关键名词是否在源文档中出现）
  · 低可信警告标记
```

### 2.4 反馈层

```
用户评价 → SQLite 存储 → (积累 N 条后/手动触发) → 参数优化算法 → 更新 RAG 参数
```

| 待定 | 选项 |
|------|------|
| 调优对象 | BM25/向量权重(α,β)、Top-K 值、重排序策略 |
| 调优算法 | 网格搜索 / 贝叶斯优化 / 遗传算法 |
| 触发方式 | 手动触发 / 自动(阈值触发) |

### 2.5 数据层

| 子模块 | 存储 | 职责 |
|--------|------|------|
| 知识图谱 | Neo4j | 实体关系持久化、Cypher 查询 |
| 向量库 | Chroma | 文档切片 + embedding 存储和检索 |
| 对话历史 | SQLite / JSON | 按会话组织，支持分页查询 |
| 上下文管理 | 内存 | 当前会话的多轮对话上下文窗口 |

### 2.6 工具层

| 工具 | 输入 | 输出 |
|------|------|------|
| 文档解析 | Word/PDF/TXT | 纯文本 |
| 知识抽取 | 纯文本 | 候选三元组（现象-原因-步骤） |
| 数据清洗 | 原始数据 | 标准化数据（去重、格式统一） |
| 数据导出 | — | zip（Neo4j + Chroma + SQLite + 配置） |
| 数据导入 | zip | 合并后的系统数据 |

## 二、文本预处理（输入到系统之前必做）

### 2.1 中文分词

中文没有空格分隔，必须用分词工具切开。本项目推荐 **jieba**：

```python
import jieba
text = "电动机不能起动伴有嗡鸣声"
words = jieba.lcut(text)
# ['电动机', '不能', '起动', '伴有', '嗡鸣', '声']
```

**本项目用在哪**：用户输入问题后，先分词再检索，提高匹配精度。

### 2.2 文本清洗

```python
# 统一小写、去标点、去多余空格
import re
def clean(text):
    text = text.lower().strip()
    text = re.sub(r'[，。！？、；：""''（）\n]', ' ', text)
    return re.sub(r'\s+', ' ', text)
```

## 三、文本表示（文字 → 数字向量）

### 3.1 TF-IDF（本项目推荐用于混合检索）

核心思想：词在文档中出现越频繁 (TF) 且在整个语料中越罕见 (IDF)，该词越重要。

> **TF（词频）**：词在当前文档中出现次数 ÷ 文档总词数。出现越频繁，TF 越高。  
> **IDF（逆文档频率）**：log(语料文档总数 ÷ 包含该词的文档数)。"的""是"等通用词 IDF 很低；"短路环""灭弧罩"等专业术语 IDF 很高。  
> **TF-IDF = TF × IDF**：在本文档频繁出现、但在其他文档罕见的词获得最高分——即真正有区分度的关键词。

```python
from sklearn.feature_extraction.text import TfidfVectorizer

corpus = [
    "电动机不能起动 有嗡鸣声",
    "线圈过热烧损 冒烟",
    "接触器不释放 卡阻"
]
vectorizer = TfidfVectorizer()
tfidf_matrix = vectorizer.fit_transform(corpus)
# 得到 3×N 的矩阵，每行是一个文档的向量
```

**本项目用在哪**：BM25 检索（关键词匹配）的背后原理就是 TF-IDF 的变体。

### 3.2 Word2Vec / BERT 向量

- **Word2Vec**：把每个词映射为一个固定长度向量（如100维），语义相近的词向量也相近
- **BERT**：整句向量，考虑上下文。本项目选 `paraphrase-multilingual-MiniLM-L12-v2` 做语义检索

> **BERT 是什么**：Google 2018 年发布的预训练语言模型。核心创新是**双向阅读**——传统模型只能从左到右读句子，BERT 同时从左右两个方向理解上下文，所以能区分"苹果很好吃"和"苹果发布新手机"中两个"苹果"的不同含义。实际使用不需要从头训练，HuggingFace 上直接下载预训练好的模型即可。本项目选的 MiniLM 是 BERT 的轻量多语言变体，能把任意中文句子转成 384 维向量，用于向量检索。

**本项目用在哪**：向量检索——将用户问题转成向量，在 Chroma 中找最相似的故障现象向量。

### 3.3 混合检索 = BM25 + 向量

```
用户输入 "电动机起不来"
  ↓
BM25关键词匹配 → 优先匹配"电动机""起动"（精确）
  +
向量语义匹配 → 也匹配"无法起动""不能转动"（同义）
  ↓
加权融合取 Top-5
```

## 四、文本相似度计算（匹配用户问题与知识库）

### 4.1 余弦相似度

> **为什么用余弦**：余弦衡量两个向量的**方向**差异，不看长度。两个文本转成向量后，方向一致（夹角≈0°）→相似度为 1，方向垂直（≈90°）→无关，方向相反（≈180°）→完全对立。比欧氏距离更适合文本——"电动机不能起动"和"电动机不能起动伴有嗡鸣声且熔断器跳闸"说的是同一件事，但后者词量大、向量更长，欧氏距离会误判为差异很大；余弦只看方向不看长度，能正确识别为相似。

```python
from sklearn.metrics.pairwise import cosine_similarity

# 用户问题向量 vs 知识库中每条故障现象的向量
similarity = cosine_similarity(user_vector, knowledge_vectors)
# 值越接近 1 越相似
```

**本项目用在哪**：向量检索的核心算法——计算用户问题与 Chroma 中每条故障现象描述的相似度。

### 4.2 相似度方法选择指南

| 文本长度 | 推荐方法 | 本项目场景 |
|---------|---------|-----------|
| 短文本（<50字） | BERT 句向量 | 用户输入的问题 |
| 中等文本 | TF-IDF + 余弦 | 故障现象描述匹配 |
| 长文本 | Doc2Vec 或分段 | 教材原文段落检索 |

## 五、命名实体识别（NER）与关系抽取

### 5.1 NER

自动识别文本中的特定实体。例如：
```
"接触器的短路环断裂导致噪声大"
 → 设备实体: 接触器
 → 部件实体: 短路环
 → 现象实体: 断裂, 噪声大
```

> **通俗理解**：NER 就是从一句话里找出"谁""什么""哪里"这些有名字的东西。方法有三——规则匹配（列词表，快但漏）、统计模型（需标注数据训练）、BERT 类模型（读上下文理解，效果最好但重）。本项目知识图谱已结构化，NER 不是必须；但可做入口层增强：用户说"我听到嗡嗡声"，NER 抓出"嗡嗡声"→匹配 Symptom 的 `primary_features` →定位"交流电磁铁噪声大"。

**本项目用在哪**：
- 用户输入模糊描述时，用 NER 提取关键词（设备名/现象描述）
- 知识图谱已有结构化实体，但 NER 可以做入口层的语义解析

### 5.2 关系抽取

从非结构化文本中提取"实体→关系→实体"三元组：
```
"电源电压过低导致接触器吸力不足"
 → (电源电压, 导致, 接触器吸力不足)
```

**本项目用在哪**：文档上传解析模块（第3周）——从 Word/PDF 中自动抽取候选故障知识三元组。

## 七、与 RAG 系统的对应关系

```
用户输入: "电动机冒烟了"
  ↓
[文本预处理] jieba分词 + 清洗
  ↓
[混合检索] BM25(关键词) + 向量(语义) → Top-5 相关 Symptom
  ↓
[知识图谱查询] Symptom → CAUSED_BY (priority排序) → Cause → FIXED_BY → Step
  ↓
[上下文组装] 检索到的 Cause + Step + SparePart + Tool + source_page → Prompt
  ↓
[LLM 生成] → 流式返回答案 + 引用章节
```

## 八、关键原则

1. **中文分词必须先做**：用户输入不经过分词直接检索，匹配率断崖下降
2. **混合检索优于单一方法**：BM25 保证精确匹配（"电动机"就是"电动机"），向量保证语义匹配（"冒烟"≈"过热烧损"）
3. **向量模型选多语言版本**：`paraphrase-multilingual-MiniLM-L12-v2` 支持中文，且模型小（118MB），适合离线部署
4. **NER 是可选增强项**：当前知识图谱已结构化，NER 作为语义入口锦上添花，但不是必须
